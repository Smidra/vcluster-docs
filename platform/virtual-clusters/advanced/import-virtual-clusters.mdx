Virtual clusters created through the vCluster CLI can be add into any project. By adding a virtual cluster into the a project, 
a `VirtualClusterInstance` resource will automatically be created in the specified project.

<Flow id="add-virtual-cluster">
  <Step>
    <FragmentConnectPlatform />
  </Step>
  <Step>
    <FragmentPlatformAddCluster />
  </Step>
  <Step>
    <FragmentPlatformConnectCluster />
  </Step>
  <Step>

    ```bash
    vcluster platform add vcluster <VIRTUAL_CLUSTER_NAME> \
      --cluster <UI_CLUSTER_NAME> \
      --namespace <VIRTUAL_CLUSTER_NAMESPACE> \
      --project <PROJECT>
      --import-name <VIRTUAL_CLUSTER_IMPORT_NAME>
    ```

    Replace the following:
    
    * `<VIRTUAL_CLUSTER_NAME>`: Name of your virtual cluster. 
    * `<UICLUSTER_NAME>`: Name of your host cluster that has the virtual cluster.
    * `<VIRTUAL_CLUSTER_NAMESPACE>`: Namespace on th host cluster that has the virtual cluster pod.
    * `<PROJECT>`: Name of the project that you want to add the virtual cluster to.
    * `<VIRTUAL_CLUSTER_IMPORT_NAME>`: Name that you want shown in the UI. 

  </Step>
  <Step>
    In order to change your kube-context back to your previous state, you'll need to run the following command:
    
    ```
    vcluster disconnect
    ```
  </Step>
</Flow>
